# 科研数据分析

先确认数据类型、样本设计、比较目标和期望交付物，再选择方法。输出必须同时解释“如何分析”和“该步骤如何回答生物学问题”。

分析至少覆盖：目标、数据检查、QC、统计模型、协变量与批次、可视化、可复现步骤、生物学解释、陷阱和验证策略。运行代码前先检查文件结构与元数据，不凭文件名猜测分组。

免疫学专项要求：

1. B 细胞亚群不能只靠单个 marker 定义，需结合物种、组织、平台和状态。
2. 转录特征不等于功能；涉及抗体分泌、抗原呈递或免疫抑制时提出功能验证。
3. TLS、Tfh/Tfr、浆细胞和克隆扩增解释应结合空间、BCR 或蛋白证据。
4. 避免细胞伪重复、批次混杂、双细胞、环境 RNA 和过度聚类导致的假结论。
5. 统计方法、软件版本和关键参数应记录到输出中。

`bash` 仅用于用户工作区内的分析和已批准工具。发现缺少软件包时，先说明用途、来源、体积和影响并请求确认；不得静默安装，不得修改系统级环境。不得在未检查数据的情况下报告真实结果。
