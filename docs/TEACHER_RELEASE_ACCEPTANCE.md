# 教师版发布验收

本文件记录 3.4「安装包与真实教师验证」的可重复验收范围。测试材料均为去标识、人工构造的医学免疫学教学与科研样例，不包含患者隐私。

## 自动验收范围

- Python 环境：无 Python、系统 Python、应用私有 Python、私有环境损坏。
- 安装修复：取消修复时保留原环境；损坏后可重新安装并恢复为可用状态。
- 教学材料：以“B 细胞活化与生发中心反应”本科教学材料生成可编辑 PPTX、逐页讲稿 DOCX、项目文件和视觉质检报告。
- bulk 数据：表达矩阵与样本元数据读取、明确分组与比较方向、可复现记录和输出清单初始化。
- 单细胞下游数据：marker 表与细胞群元数据读取、分析扩展预检、转录特征解释边界和任务恢复。
- 任务恢复：分析中断后保留材料、分组信息和最近成功步骤，从该步骤之后继续。
- Windows 成品：安装程序、主程序、Kun 运行组件、SQLite 原生依赖、内置课件与科研分析能力、文件大小及 SHA-256。

## 教师桌面冒烟验收

Windows 成品应用需直接启动并检查：

1. 工作台与教学、科研、能力中心入口正常显示。
2. “科研数据分析”无需终端即可进入。
3. 未安装私有分析引擎时，显示“一键安装 / 稍后安装”；环境损坏时显示“一键修复”。
4. 页面不向老师暴露 PATH、pip、venv、site-packages 或终端命令。
5. bulk 与单细胞统计任务在缺少样本分组和比较关系时不可提交。
6. 导出、失败续跑和修复均由应用内按钮完成，不要求打开开发者工具。

## 正式发布边界

`npm run dist:win` 会生成 `dist/teacher-release-verification.json`。内部 alpha 包允许未签名，但报告必须如实记录 Authenticode 状态。正式发给老师前必须配置 Windows 代码签名，并以 `ZHIYAN_REQUIRE_SIGNED=1` 重新执行发布构建；未通过有效签名校验的安装包不得作为正式公开版本分发。
