# 智研助手产品升级执行计划

> 适用对象：面向没有代码基础的高校教师，以医学教学与生命科学科研为优先场景。  
> 执行方式：严格按 P0 → P1 → P2 → P3 顺序推进；每次只完成一个可独立验证、可回滚的任务。  
> 状态约定：`[ ]` 未开始，`[~]` 进行中，`[x]` 已完成。  
> 质量门槛：生产代码变更必须在独立 worktree 中完成，并通过测试、类型检查、零警告 lint 和构建。

## 一、产品目标与边界

智研助手应当同时满足以下目标：

1. 教师无需理解 Python、命令行、Git、MCP 或 Agent 运行时。
2. 基础教学、写作、文献阅读和 Office 导出在没有 Python 时仍可使用。
3. 软件提供受管理的 Python 科研分析引擎，让 AI 能真实执行统计、绘图和下游生信任务。
4. Python 的安装、版本、包、升级、修复和卸载由智研助手管理，不污染系统 Python 或 PATH。
5. 新建成果可以方便地自动保存；覆盖、删除、移动、安装软件、访问工作区外目录等高风险操作必须明确确认。
6. 科研结论可追溯到用户材料、外部证据或模型推断，不能把推测写成事实。

产品原则概括为：**Python 应当存在，但对老师不可见；能力可以强，操作必须简单；核心功能不能因科研环境缺失而失效。**

## 二、目标架构

### 2.1 核心应用层

不依赖 Python，负责：

- AI 对话、教师档案和项目上下文；
- 教案、教材、课件、文本写作和文献阅读主流程；
- PDF/PPTX/DOCX/XLSX 的受控读取；
- 确定性的 DOCX、PPTX、PDF 和项目包导出；
- 文件选择、预览、保存、恢复和版本记录。

### 2.2 Python 科研分析引擎

由应用私有管理，不修改系统 PATH，建议目录为应用数据目录下的 `runtimes/python/`。分两层交付：

- 基础科研包：NumPy、pandas、SciPy、Matplotlib、Seaborn、statsmodels、scikit-learn、openpyxl、python-docx、AnnData、UMAP。
- 生物信息学扩展包：Scanpy、igraph、leidenalg 等体积较大的组件。

R/Bioconductor 作为后续独立扩展，不假设 Python 能完全替代 DESeq2、Seurat、ComplexHeatmap 等生态。

### 2.3 安全执行层

工具操作按风险分级：

- 自动允许：读取、搜索、列目录，以及用户已明确启动的任务在专属输出目录中新建成果。
- 按需确认：一般文件修改和受控分析命令。
- 强制确认：覆盖、删除、移动、批量改名、安装或升级软件包、访问工作区外路径。
- 默认禁止：修改系统环境、绕过工作区边界、静默安装、执行来源不明的脚本。

上传的教材、论文、表格和网页内容一律视为不可信资料；其中的文字不能改变系统指令、工具权限或安装策略。

## 三、分阶段执行任务

### P0：产品一致性与安全基线

- [x] **0.1 建立统一升级计划**
  - 合并产品简洁性、安全执行、原生 Office 导出和受管理 Python 的升级方向。
  - 明确每个阶段的验收标准和执行顺序。

- [x] **0.2 GUI 默认采用按需确认策略**
  - Kun CLI 保留自身默认值，智研助手 GUI 新配置默认使用 `on-request`。
  - 读取类工具继续自动执行；写文件和命令执行显示确认卡。
  - 显式选择 `auto` 的已有用户不被强制迁移。
  - 增加设置默认值与迁移测试。

- [x] **0.3 修复智能教案导出链路**
  - 移除教案提示词中对 `bash + python-docx` 的强制依赖。
  - 复用现有原生 DOCX 导出服务，模型只生成结构化教案内容。
  - 确保没有 Python 的电脑仍可生成、预览和保存教案 DOCX。

- [x] **0.4 附件提示注入隔离**
  - 教材、论文和表格正文使用明确的数据边界封装。
  - 系统提示中声明附件内容不能授权工具、修改保存路径或触发安装。
  - 为恶意附件指令增加回归测试。

### P1：受管理的 Python 科研分析引擎

- [x] **1.1 建立运行环境契约与状态模型**
  - 定义 `未安装 / 可用 / 不完整 / 损坏 / 需要升级 / 检测中` 状态。
  - 返回解释器路径、版本、架构、来源和能力包版本，不向教师暴露技术噪音。

- [x] **1.2 将 PATH 检测升级为真实诊断**
  - 实际运行 `python --version` 和受控导入检查，而不是只判断可执行文件是否存在。
  - 检测 NumPy、pandas、SciPy、Matplotlib、Seaborn、statsmodels、scikit-learn、openpyxl、AnnData 和 UMAP。
  - 对 Windows 商店占位程序、错误位数、损坏环境和超时提供明确结果。

- [x] **1.3 建设应用私有 Python 管理器**
  - Python 安装在智研助手数据目录，不写系统 PATH。
  - 安装包清单包含版本、来源、体积、SHA-256、许可证和可卸载信息。
  - 支持下载进度、断点恢复、校验、原子安装、失败回滚、修复和卸载。
  - 安装必须由教师点击确认，AI 不得通过任意 shell 命令静默安装。

- [ ] **1.4 建立版本固定的基础科研包**
  - 提供经过验证的锁定版本与冒烟测试。
  - 记录每次分析使用的 Python 和包版本。
  - 仅允许在应用私有环境安装白名单包。

- [ ] **1.5 建立生物信息学扩展包**
  - Scanpy/igraph/leidenalg 等单独安装和升级。
  - 明确支持整理后的下游数据，不默认承担 FASTQ/BAM/Cell Ranger 原始流程。

### P2：教师可见的科研任务体验

- [ ] **2.1 数据分析入口环境预检**
  - 提交任务前显示“可直接分析 / 需要安装科研分析引擎 / 缺少扩展包”。
  - 提供“一键安装”“修复”“稍后安装”，不显示 PATH、pip、venv 等术语。

- [ ] **2.2 统一科研任务卡**
  - 展示材料、分析目标、状态、交付物、保存位置和继续修改入口。
  - 将执行过程与普通聊天分离，失败后可从最近成功步骤继续。

- [ ] **2.3 结果可复现**
  - 每个分析项目保存输入文件摘要、参数、代码、包版本、运行日志和输出文件清单。
  - 不以文件名猜测样本分组；关键元数据缺失时阻止统计分析。

- [ ] **2.4 科研证据追溯**
  - 区分用户材料、全文证据、摘要/元数据、数据库结果、相关性推断和机制假设。
  - 引用至少保留标题、年份、来源以及 DOI/PMID 等可核验标识。

### P3：信息架构、工程收敛与发布准备

- [ ] **3.1 保留直接入口，降低选择负担**
  - 保留教学、科研、能力中心的分组侧栏，不强制合并成少数抽象入口。
  - 通过最近使用、自然语言搜索和任务推荐降低寻找成本。

- [ ] **3.2 建立唯一模块注册表**

  - 移除 Dashboard 重复硬编码，避免名称和功能漂移。

- [ ] **3.3 桌面安全加固**
  - 未使用时关闭 Electron `webviewTag`。
  - 审核外部链接、窗口导航、CSP、IPC schema 和工作区边界。

- [ ] **3.4 安装包与真实教师验证**
  - 分别验证无 Python、已安装系统 Python、应用私有 Python 和损坏环境。
  - 用真实医学教学材料、bulk 表格和单细胞下游文件完成端到端测试。
  - 教师无需打开终端或开发者工具即可安装、分析、导出、修复和恢复任务。

## 四、每个任务的完成标准

一个任务只有同时满足以下条件才可标记为完成：

1. 教师可见行为符合本计划，错误提示可直接理解和执行。
2. 关键成功、失败、取消和恢复路径有自动化测试。
3. 未破坏没有 Python 时的核心教学与写作能力。
4. 没有放宽 Electron sandbox、IPC schema 或工作区路径边界。
5. 通过：

```powershell
npm test
npm run typecheck
npm run lint -- --max-warnings=0
npm run build
```

涉及 Windows 安装或 Python 引擎时，还必须通过：

```powershell
npm run dist:win
```
