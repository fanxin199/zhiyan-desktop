# AI 能力与内置 Skills 架构

## 产品原则

智研助手不是固定模板集合，而是“教师可见的简单任务入口 + 后台 AI 能力编排”。

- 教师通过对话、按钮和上传材料描述目标。
- Kun 根据提示词、命令和附件类型自动激活相关 Skill。
- Skill 提供专业工作流、证据规则、输出标准和工具白名单。
- 模型继续负责理解上下文、规划步骤、调用工具和生成结果。
- 关键文件操作、命令执行和软件安装仍受工作区边界与审批策略控制。

## 内置能力

首版随安装包提供 8 个第一方 Skills：

1. 教学课件设计
2. 智能教案
3. 题库与评价设计
4. 论文与基金写作
5. 文献检索与综述
6. 科研数据分析
7. 科研绘图与机制图
8. 文档整理与转换

源文件位于 `resources/builtin-skills/`。安装包将其放入
`resources/builtin-skills`，应用启动时再同步到 Kun 数据目录下的
`skills/builtin/`。每个 Skill 使用 `skill.json` 声明触发条件、版本和允许工具。

版本变化时只替换同名的受管理内置 Skill。项目 Skill、用户 Skill 和其他目录不会被删除或覆盖。

## 与个人全局 Skills 的关系

产品可以参考开发者电脑上的优秀 Skills，但不得直接把个人全局目录整体打包：

- 个人目录不是稳定的产品依赖。
- 第三方内容可能缺少明确再分发许可。
- 其中的脚本可能依赖本机路径、账号、商业软件或私有数据。
- 面向开发者的 Skill 不适合无编码基础教师。

因此首版采用第一方重写。用户仍可在高级扩展中加载项目或个人 Skills。

## 工具与依赖

Skill 可以通过 `dependencies.json` 声明可选命令。能力中心只检测 `PATH`，不会执行命令。

- 已安装工具显示“可用”。
- 缺失工具显示“缺少可选工具”，基础工作流仍可继续。
- AI 需要安装工具时，必须先说明用途、来源和影响。
- 用户确认后，Kun 仍会按审批策略确认 `bash` 调用。
- 不允许静默安装、修改系统级环境或绕过工作区边界。

## 权限边界

- `read/grep/find/ls`：读取和检查工作区材料。
- `write/edit`：生成或修改工作区成果，执行时需要文件变更审批。
- `web_fetch/web_search/MCP`：用于文献与事实核实，受联网配置控制。
- `bash`：仅数据分析、绘图和格式转换 Skill 按需开放，执行受审批。
- Skill 白名单不能扩大 Kun 原有权限，只能进一步限制当轮可调用工具。

## 后续扩展

- 为 Office、统计和绘图任务增加更多内置确定性工具，减少对系统软件的依赖。
- 建立带许可证、签名、版本和安全扫描的 Skill 包格式。
- 增加 Skill 执行记录、依赖安装向导和一键诊断。
- 将课程项目中的课件、教案、题库和资料统一为可复用上下文。
