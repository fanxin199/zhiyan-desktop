# 科研工作台与内置能力规划

## 产品定位

科研模块面向没有编程经验的医学与生命科学教师。主界面只呈现教师任务，不暴露 Skill、MCP、Python、R、命令行等技术概念。

一级任务分为：

- 文本写作：自然基金、论文写作与润色、综述、长文档分段修改。
- 文献阅读：文献精读、关键图解读、文献汇报 PPT、主题证据整理。
- 下游数据分析：整理后 bulk mRNA 与单细胞数据的可视化、解释和报告。
- 科研绘图：统计图、机制图、BioRender/PPT/Illustrator 绘图说明。

## 精选内置能力

全局 skills 不整体打包。只吸收适合教师主流程、依赖可控、许可证可审查的能力，并重写为智研助手第一方内置能力。

- 微信发布、Obsidian 自动化、NotebookLM、前端设计、插件开发类能力。

- primer design、deepTools、gtars 等低频高级科研能力，后续可放在高级扩展。

## 上下文写作系统

写作模块必须避免“只看当前段落就改写”。所有长文本任务默认使用项目主线记忆。

每个写作项目至少保存：

- `blueprint.md`：项目题目、科学问题、核心假说、研究目的、关键数据、术语表、章节边界。
- `status.md`：当前进度、已完成章节、下一步目标、最近一次用户修改意见。
- `sections/`：待写或待改写的章节。
- `cards/`：每个已完成章节的摘要卡片。
- `outputs/`：最终文本、导出文件和修改说明。

执行原则：

1. 先建立或读取 Blueprint，再写正文。
2. 每次只处理一个章节或小节，除非用户明确要求快速草稿。
3. 写作前注入项目主线、本节目标、上一节摘要、下一节预期和术语表。
4. 写作后生成 Section Card，记录核心论点、证据、关键术语、尾句承接点和待确认问题。
5. 对基金和论文中的非用户实验结果论点，优先检索并补充 PMID/DOI；无法核实时明确标注。
6. 不擅自增加实验结果、样本量、统计显著性、伦理批准号或不存在的参考文献。

## 下游数据分析边界

数据分析模块默认从整理好的数据开始，不做原始测序流程。

bulk mRNA 首批支持：

- 表达矩阵、样本分组表、差异分析结果表、富集结果表、基因列表。
- PCA、样本相关性热图、火山图、差异基因热图、GO/KEGG/GSEA、重点免疫基因图。

单细胞首批支持：

- h5ad、细胞注释表、marker 表、差异表达表和整理后的表达矩阵。
- UMAP/tSNE、细胞比例图、marker dotplot、violin、热图、通路评分和亚群比较。

免疫学解释要求：

- B 细胞亚群不能只由单个 marker 定义。
- 转录特征不等于功能结论。
- TLS、浆细胞、Tfh/Tfr 和克隆扩增解释应结合空间、BCR 或蛋白证据。
- 每个结果都说明能支持什么、不能支持什么、还需要什么验证。

## 能力中心原则

能力中心负责显示内置能力、外部连接和高级配置状态。普通教师主流程中不应出现 MCP JSON、Skill 文件路径或命令行配置。

MCP 在教师侧显示为“外部资源连接”，例如 PubMed、Zotero、Google Drive 或数据库连接。高级设置中才显示 MCP 配置、信任范围和诊断信息。
