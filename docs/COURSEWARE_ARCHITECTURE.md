# 课件素材工作流

## 数据流程

```text
PDF/PPTX
  -> 文字与图片分析
  -> 重复/装饰图片过滤
  -> 教师审核
  -> 教学蓝图与逐页内容
  -> 图片自动匹配
  -> 教师逐页调整
  -> PPTX + DOCX + .zhiyan-courseware
```

## 图片提取

### PDF

- PDF.js 读取文字、页面尺寸和图像绘制操作。
- 候选区域按页面坐标保存，渲染进程只裁剪有意义的大图。
- 扫描版 PDF 若无可搜索文字，明确提示先做 OCR。

### PPTX

- PPTX 按 Office Open XML 压缩包解析，不依赖 PowerPoint 或 Python。
- 读取幻灯片文字、图片关系、原始媒体和图片位置。
- 按媒体内容去重；高频重复、小尺寸和低面积图片默认标记为装饰图。
- 原始图片以 data URL 进入审核界面，导出项目时独立写入 `assets/`。

## 项目格式

Project v2 保存：

- 原教材类型、路径、页数和可搜索状态
- 教学任务、蓝图、逐页内容和证据
- 图片来源页、审核状态、图注、尺寸和分配关系

`.zhiyan-courseware` 是 ZIP 容器，包含 `project.json` 和 `assets/`。旧版 JSON 项目在读取时自动迁移到 v2。

## 与全局 PPT Skill 的关系

当前产品参考了本机 `pptx` 与 `baoyu-slide-deck` skill 的成熟原则：

- 从 `ppt/media` 读取原始图片
- 一页一个主要视觉锚点
- 图片保持比例并根据横竖构图选择布局
- 医学课堂优先大图、低文字密度和可投影阅读
- 导出后必须进行内容与视觉检查

产品不会依赖开发者电脑的个人全局 skill 路径。课件能力已重写为智研助手第一方
`courseware-builder` Skill，并随安装包分发；原生课件流程仍可独立完成提取、审核、
匹配和导出。完整机制见 `AI_SKILL_ARCHITECTURE.md`。
